Capítulo 6: Bucles while

La técnica de definir nuevos métodos y definir bucles for, por más útiles que sean, en realidad no permite que Karel resuelva ningún problema nuevo. Cada vez que ejecutas un programa, siempre hace exactamente lo mismo. Los programas se vuelven mucho más útiles cuando pueden responder de manera diferente a diferentes entradas.

Como ejemplo. Digamos que querías escribir un programa para tener a Karel moviéndose en una pared. Pero no solo desea que este programa funcione en un mundo con un tamaño fijo. Te gustaría escribir un solo programa que podría funcionar en cualquier mundo.

Bucle básico while

En Karel, un while de bucle se utiliza para repetir un cuerpo de código como-largo como una condición dada se mantiene. El bucle while tiene la siguiente forma general:

while(prueba){
declaraciones a repetir
}

El flujo de control de un bucle while es el siguiente. Cuando el programa llega a un bucle while comienza a repetir un proceso en el que primero verifica si la prueba pasa, y si es así, ejecuta el código en el cuerpo.

Cuando el programa verifica si la prueba pasa, decide si la prueba es verdadera para el estado actual del mundo. Si es así, el bucle ejecutará el código en el cuerpo. Si la prueba falla, el ciclo termina y el programa continúa.

Cuando el programa ejecuta el cuerpo del bucle, el programa ejecuta las líneas en el cuerpo una a la vez. Cuando el programa llega al final del bucle, vuelve a la parte superior del bucle. Luego vuelve a verificar la prueba, y continúa en bucle si pasa. El programa no sale del bucle hasta que llega a una comprobación y la prueba falla.

Karel tiene muchas afirmaciones de prueba, y las analizaremos todas en el siguiente capítulo. Por ahora vamos a usar una sola declaración de prueba: elFrenteEsClaro() que es verdadera si no hay un muro directamente enfrente de Karel.

Eso se ve genial. Excepto por un problema. En todos los mundos, Karel no coloca una señal acústica en el último cuadrado de la línea (mira detenidamente). Cuando Karel está en la última casilla, el programa no ejecuta el cuerpo del bucle porque la prueba ya no pasa, Karel se enfrenta a una pared. Puede sentirse tentado de intentar cambiar el orden del cuerpo para que Karel se mueva antes de colocar una señal acústica. El código es editable así que ¡pruébalo!

Hay un problema más profundo que ninguna reorganización del cuerpo puede resolver. Para el mundo con 7 columnas, Karel necesita poner 7 beepers, pero solo debe moverse 6 veces.
