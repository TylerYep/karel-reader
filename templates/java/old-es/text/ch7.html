Capítulo 7: Si las declaraciones

La construcción de flujo de control de programación final para aprender son sentencias condicionales ( if y if/else).

Condicionales basicas

Una instrucción if/else ejecuta un bloque de código "if" si, y solo si, la prueba proporcionada es verdadera para el estado del mundo en el momento en que el programa llega a la instrucción. De lo contrario, el programa ejecuta el bloque de código "else".

if(prueba){
si el código de bloqueo
} else{
otro bloque de código
}

Para tener una idea de dónde pueden ser útiles las declaraciones condicionales, escribamos un programa que haga que Karel invierta una línea de señales acústicas. Si un cuadrado tenía un beeper anteriormente, Karel debería recogerlo. Si un cuadrado no tiene señal acústica, Karel debería poner uno abajo.

Tenga en cuenta que un if declaración no se necesita tener una else bloquear - en cuyo caso los opperates declaración como un while de bucle que sólo se ejecuta una vez:

if(prueba){
si el código de bloqueo
}

Condiciones

Ese último ejemplo utilizó una nueva condición. Aquí hay una lista de todas las condiciones que Karel conoce:

Prueba Opuesto Lo que verifica
elFrenteEsClaro()elFrenteEstaBloqueado()¿Hay un muro delante de Karel?
beepersPresente()noHayBeepersPresentes()¿Hay beepers en esta esquina?
izquierdaEsClara()izquierdaEstáBloqueada()¿Hay un muro a la izquierda de Karel?
laDerechaEsClara()derechoEstáBloqueado()¿Hay un muro a la derecha de Karel?
beepersEnBolsa()noHayBeepersEnLaBolsa()¿Hay algún beepers en la bolsa de Karel?

Poniendolo todo junto

Felicidades Ahora conoces todos los bloques de flujo de control de programación básicos. Mientras los aprendió con Karel, los métodos, bucles while que, bucles for, si / else sentencias funcionan de la misma manera en casi todos los lenguajes principales, incluido Java.

Ahora que tiene los componentes básicos, puede juntarlos para construir soluciones a problemas cada vez más complejos. En gran medida, la programación es la ciencia de resolver problemas por computadora. Debido a que los problemas a menudo son difíciles, las soluciones y los programas que implementan esas soluciones también pueden ser difíciles. Para facilitar el desarrollo de esas soluciones, debe adoptar una metodología y disciplina que reduzca el nivel de esa complejidad a una escala manejable.
