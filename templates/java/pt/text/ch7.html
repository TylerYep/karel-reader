Capítulo 7: Se Declarações

A construção final do fluxo de controle de programação principal para aprender são instruções condicionais ( if e if/else).

Condicionais Básicos

Uma instrução if/else executa um bloco de códigos "if" se e somente se o teste fornecido for verdadeiro para o estado do mundo no momento em que o programa atinge a instrução. Caso contrário, o programa executa o bloco de códigos "else".

if(test){
se bloco de código
} else{
outro bloco de código
}

Para ter uma ideia de onde declarações condicionais podem ser úteis, vamos escrever um programa que faça Karel inverter uma linha de sinais sonoros. Se um quadrado tivesse um bip, Karel deveria pegá-lo. Se um quadrado não tiver bip, Karel deve derrubá-lo.

if(test){
se bloco de código
}

Condições

Esse último exemplo usou uma nova condição. Aqui está uma lista de todas as condições que Karel conhece:

Teste Oposto O que verifica
frenteÉClaro()frenteEstáBloqueada() Há uma parede na frente de Karel?
beepersPresent()nãoHáBipersPresentes() Há beepers neste canto?
leftIsClear()esquerdaEstáBloqueada() Existe um muro à esquerda de Karel?
certoEstáClaro()certoEstáBloqueado() Existe um muro para o Karel, certo?
bipesNoSaco()nãoBeepersInBag() Algum apito na bolsa de Karel?
voltadoParaONorte()nãoVoltadoParaONorte() Karel está voltado para o norte?
viradoASul()nãoVoltadoParaOSul() Karel está voltado para o sul?
voltadoParaOLeste()nãoVoltadoParaOLeste() Karel está voltado para o leste?
deFrenteParaOOeste()nãoVoltadoParaOOeste() Karel está voltado para o oeste?

Colocando tudo junto

Parabéns! Agora você conhece todos os principais blocos de fluxo de controle de programação. Enquanto você os aprendeu com Karel, métodos, while loops, for loops, instruções if / else funcionam da mesma maneira em quase todas as principais linguagens, incluindo Java.

Agora que você tem os blocos de construção, você pode colocá-los juntos para criar soluções para problemas cada vez mais complexos. Em grande medida, a programação é a ciência de resolver problemas por computador. Como os problemas costumam ser difíceis, as soluções - e os programas que implementam essas soluções - também podem ser difíceis. Para facilitar o desenvolvimento dessas soluções, você precisa adotar uma metodologia e disciplina que reduza o nível dessa complexidade a uma escala gerenciável.
