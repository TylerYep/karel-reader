Olsa bile IlkKarel Yukarıdaki program gerçekleştirmek mümkün olduğunu göstermektedir turnRight Sadece Karel'in yerleşik komutlarını kullanarak yapılan operasyonda, ortaya çıkan program kavramsal olarak açık değildir. Programın zihinsel tasarımında, Karel çıkıntının üstüne ulaştığında sağa döner. Bunu yapmak için üç turnLeft komutunu kullanmanız gerçektir. Sonuçta ortaya çıkan program sadece daha kısa ve yazması kolay olmayacak, aynı zamanda okuması da daha kolay olacaktır.

Yeni Komutları Tanımlama

Neyse ki, Karel programlama dili, yeni komutları yeni yöntem tanımları ekleyerek tanımlamayı mümkün kılar. Ne zaman yararlı işler yapan bir dizi Karel komutunuz varsa - örneğin sağa dönme - bu komut dizisini uygulayan yeni bir yöntem tanımlayabilirsiniz. Yeni bir Karel metodu tanımlamak için kullanılan format, tanımının aynısıdır. run önceki örneklerde, bu kendi başına bir yöntem tanımıdır. Tipik bir yöntem tanımı şöyle görünür:

   private void isim () {
Yöntemin gövdesini oluşturan komutlar
   }

Bu düzende ad, yeni yöntem için seçtiğiniz adı temsil eder. Tanımı tamamlamak için tek yapmanız gereken, küme parantezleri arasındaki satırlardaki komut dizisini sağlamaktır. Örneğin, tanımlayabilirsiniz turnRight aşağıdaki gibi:

   private void turnRight() {
turnLeft();
turnLeft();
turnLeft();
   }

Benzer şekilde yeni bir tanım tanımlayabilirsiniz. turnAround Bunun gibi bir yöntem:

   private void turnAround() {
turnLeft();
turnLeft();
   }

Yeni bir yöntemin adını, Karel'in yerleşik komutlarından herhangi biri gibi kullanabilirsiniz. Örneğin, bir kez tanımladıysanız turnRight , üç turnLeft komutunu KarelHızlandırmak program tek bir çağrı ile turnRight yöntem. İşte kullanan programın gözden geçirilmiş bir uygulaması turnRight :

<% include('templates/java/'+lang+'/programs/runnable.html', name="stepUp2", world="stepUp") %>

Tabii ki, tanımları arasında belirgin bir fark var. run ve turnRight Yukarıdaki programdaki yöntemler: run yöntem olarak işaretlendi public kıyasla turnRight olarak işaretlenmiş private . Bu iki atama arasındaki fark kamu yöntemlerinin program dışından çağrılabilirken özel yöntemlerin kullanılmamasıdır. run yönteminin halka açık olması gerekiyor, çünkü Karel ortamının run için run komutu vermesi gerekiyor. Tanımları mümkün olduğu kadar gizli tutmak genellikle iyi bir programlama uygulamasıdır.

Yöntem Kod Blokları

İki küme parantezi arasında bir grup komut { } denir kod bloğu . Programınızın gövdesi, bir yöntemin gövdesi gibi bir kod bloğudur. Bir kod bloğunun içeriğinin tümünde bir sekme girintisi olduğuna dikkat edin. Bu, stilistik olarak önemlidir, çünkü insanların belirli bir satırda hangi kod satırlarının olduğunu bilmelerini kolaylaştırır.

İstediğiniz kadar yöntem tanımlayabilirsiniz. Hepsi program kod bloğunuzun içinde birbiri ardına yazılmalıdır. Başka bir yöntemin içindeki bir yöntemi tanımlayamazsınız.
