Chapitre 7: Les déclarations if

La dernière structure de flux de contrôle de programmation à apprendre est des instructions conditionnelles ( if et if/else ).

Conditions de base

Un if/else Cette instruction exécute un "si" code-block si et seulement si letester est vrai pour l'état du monde au moment où le programme atteint la déclaration. Sinon, le programme exécute le bloc de code "else".

if( tester ){
si code-block
} else {
sinon code-block
}

Pour avoir une idée de l'endroit où les instructions conditionnelles peuvent jetons utiles, écrivons un programme dans lequel Karel inversera une ligne de jetons . Si un carré avait précédemment un jeton , Karel devrait le ramasser. Si un carré n'a pas jeton , Karel devrait en poser un.

Notez qu'un if déclaration n'a pas besoin d'avoir un else bloquer - auquel cas la déclaration opère comme un while boucle qui n’exécute qu’une fois:

if( tester ){
si code-block
}

Mettre tous ensemble

Félicitations! Vous connaissez maintenant tous les blocs principaux de flux de contrôle de programmation. Pendant que vous les avez apprises avec Karel, les boucle while , boucle while s, boucle for s, if / else fonctionnent de la même manière dans la plupart des langages principaux, y compris Java.

Maintenant que vous avez les éléments de base, vous pouvez les assembler pour trouver des solutions à des problèmes de plus en plus complexes. Dans une large mesure, la programmation est la science de la résolution de problèmes par ordinateur. Parce que les problèmes sont souvent difficiles, les solutions - et les programmes qui les implémentent - peuvent l'être également. Afin de faciliter le développement de ces solutions, vous devez adopter une méthodologie et une discipline qui réduisent le niveau de cette complexité à une échelle raisonnable.
